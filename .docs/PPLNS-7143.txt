CallMiner - Re-run Lambda NULL Threshold Handling

As part of the Oct 27th Execution when one of the callminer execution happened as a rerun, the same was errored out at the Landing Lambda even with the retry the error persisted, it was errored out with the exception type of play.api.libs.json.JsResultException. This happened due to the assignment of threshold value to NULL in the input passed to the execution. Which presently is not allowed in the landing lambda. Whenever the callminer re-run lambda runs with the alerts if there is no values present in the alarm, then the threshold is being used as None which is the default of the threshold as a result the callminer step function is failing, in order to tackle this issue we need to make sure we exit out of the lambda in case of threshold is none without triggering the callminer workflow which will cause an issue. {env}-ingestion-callminer-rerun-lambda Lambda function would need to be changed accordingly. More details on the same could be found in the attached analysis document that was done at the time of Production Support.
